# kbdlayout

Lightweight neural-network tool that detects whether a piece of text was typed
with the wrong keyboard layout (**Ukrainian ↔ English**) and converts it back
to the intended text.

```
"Ghbdsn, zr cghfdb?"   →  "Привіт, як справи?"
"Руддщб, рщц фку нщг?" →  "Hello, how are you?"
```

Best validation accuracy: **99.58 %** (5 epochs, ~400 000 examples).

## Python API

```python
from kbdlayout import LayoutPredictor

p = LayoutPredictor()

# Auto-detect and convert wrong-layout text
p.convert("Ghbdsn, zr cghfdb?")   # → "Привіт, як справи?"
p.convert("Руддщб, рщц фку нщг?") # → "Hello, how are you?"
p.convert("Привіт, як справи?")   # → "Привіт, як справи?"  (unchanged, correct)

# letters_only=True (default) — remaps only letter keys; punctuation is preserved as-is
p.convert("Ghbdsn, zr cghfdb?")                    # → "Привіт, як справи?"
p.convert("Ghbdsn, zr cghfdb?", letters_only=True) # → "Привіт, як справи?"  (same)

# letters_only=False — also remaps punctuation keys that differ between layouts
# On a Ukrainian layout the "," key produces "б" and "?" produces "," —
# so if *everything* was typed in the wrong layout, pass letters_only=False
p.convert("Ghbdsn, zr cghfdb?", letters_only=False) # → "Привітб як справи,"
p.convert("Ghbdsn? zr cghfdb&", letters_only=False) # → "Привіт, як справи?"
#  comma → б,  question mark → comma  (physical-key remapping)

# Detailed analysis
p.advise("Ghbdsn, zr cghfdb?")
# {
#   'wrong_layout': True,
#   'text_layout': 'en',
#   'need_layout': 'uk',
#   'confidence': 0.9998
# }

# Class probabilities
p.probs("Ghbdsn, zr cghfdb?")
# {'uk_uk': 0.0001, 'uk_en': 0.9998, 'en_en': 0.0001, 'en_uk': 0.0000}

# Per-word mode — useful for mixed-layout messages
for item in p.advise_per_word("D nt,t ' Steam?"):
    print(item)
```

### Low-level layout conversion (no model needed)

```python
from kbdlayout import en_to_uk, uk_to_en

en_to_uk("Ghbdsn")  # → "Привіт"
uk_to_en("Привіт")  # → "Ghbdsn"
```

## CLI

After installation the `kbdlayout` command is available:

```bash
# Detect layout
kbdlayout "Ghbdsn, zr cghfdb?"

# Detect and print the corrected version
kbdlayout --convert "Ghbdsn, zr cghfdb?"

# Per-word mode (for mixed-layout messages)
kbdlayout --per-word "D nt,t ' Steam?"

# JSON output
kbdlayout --json --convert "Ghbdsn, zr cghfdb?"

# Read from stdin
echo "Ghbdsn, zr cghfdb?" | kbdlayout --convert

# Interactive mode
kbdlayout

# Custom checkpoint
kbdlayout --ckpt path/to/model.pt "some text"
```

Or with `python -m`:

```bash
python -m kbdlayout --convert "Ghbdsn, zr cghfdb?"
```

## Labels

The model classifies text into 4 categories:


| Label   | Meaning                                           | Example                |
| ------- | ------------------------------------------------- | ---------------------- |
| `uk_uk` | Ukrainian typed on Ukrainian layout (**correct**) | `Привіт, як справи?`   |
| `uk_en` | Ukrainian typed on English layout (**wrong**)     | `Ghbdsn, zr cghfdb?`   |
| `en_en` | English typed on English layout (**correct**)     | `Hello, how are you?`  |
| `en_uk` | English typed on Ukrainian layout (**wrong**)     | `Руддщб, рщц фку нщг?` |


---

## How it works

A **character-level TextCNN** reads up to 64 characters of the input and
classifies it into one of the four labels above.  
The mapping between physical keys and characters on Ukrainian (Enhanced) and
US English (QWERTY) layouts is defined in `kbdlayout/layout.py`.

Training data was generated by:

1. Sampling chunks from Ukrainian and English text corpora.
2. Producing a "wrong-layout" copy of each chunk via the key mapping.
3. Training for 5 epochs on ~400 000 examples (80/20 balance between correct and wrong-layout).
